# FANZA X運用支援 管理画面

このアプリは、FANZAアフィリエイト用のX運用を支援する管理ダッシュボードです。
新しいUIは `/`、旧UIは `/legacy` で確認できます。

## 新UIの画面

- ダッシュボード: 今日の投稿状況、キュー、エラー、API接続状態を確認
- 商品検索: FANZA作品候補を検索し、サンプル動画・許可メーカー・除外理由を確認
- 投稿キュー: 投稿前候補のdry-run、本番投稿、スキップ
- dry-run確認: 投稿本文、URL、動画、Compliance、POST_ENABLED/DRY_RUN状態を確認
- 動画確認: 添付動画のプレビューと処理方針を確認
- 投稿ログ: 投稿済み/dry-run/失敗ログの検索とCSV出力
- 除外ログ: 除外理由の集計と確認
- 設定状態: Secret値を表示せず、設定済み/未設定だけを確認
- 許可/NG管理: allowed_makers.json と ng_keywords.json を管理
- ヘルプ: 1日の運用フロー、本番投稿前チェックリスト

## 安全設計

- APIキー、Xトークン、DMM/FANZA API ID、アフィリエイトIDなどのSecret値は画面に表示しません。
- 投稿アカウントはX側で「センシティブな内容を含むメディア」設定済みで運用してください。
- サムネイルと動画プレビューは初期状態でぼかし表示です。
- 本番投稿は確認モーダルを挟みます。
- `POST_ENABLED=false` または `DRY_RUN=true` の場合、本番投稿ボタンは無効化されます。
- 許可メーカー外、NGキーワード、Compliance NGの候補は投稿前に警告されます。

## Replitで確認する手順

1. Replit Secretsに必要な値を設定します。
2. アプリを起動します。
3. `/` にアクセスして新しい管理画面を開きます。
4. 「設定状態」でSecretが「設定済み/未設定」だけで表示されることを確認します。
5. 「商品検索」で候補を取得します。
6. 「投稿キュー」で `dry-runで投稿確認` を実行します。
7. 問題がない場合のみ、本番投稿モーダルを確認して投稿します。

## 旧UI

既存機能を壊さないため、旧UIは `/legacy` に残しています。
