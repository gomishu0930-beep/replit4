あなたは既存のX自動投稿・アフィリエイト運用Botの開発担当です。
このプロジェクトを読み込み、X投稿の自律運用機能を実装してください。

目的：
Xアカウントでアフィリエイト投稿を自律的に運用できるようにする。
ただし、スパム化・過剰投稿・規約違反・成人向け表現の過激化・API制限超過を避け、安全に運用できる設計にする。

前提：
- 既存プロジェクトの構成を最初に確認する
- 既存機能を壊さない
- X APIを使用する前提で実装する
- ログイン画面の自動操作、規約回避、BAN回避目的の挙動は実装しない
- APIキー、トークン、Secretsは絶対にコードへ直書きしない
- .env / Replit Secrets から読み込む
- 本番投稿数を勝手に増やさない
- 重要な変更は安全側に倒す

実装してほしい機能：

1. 自律投稿スケジューラー
- 指定時間に投稿候補を生成する
- 投稿カテゴリを分ける
  - ranking
  - new
  - sale
  - random
  - review
- 投稿頻度は設定ファイルで管理する
- 初期設定は控えめにする
- 連続投稿・短時間投稿を避けるクールダウンを実装する

2. 投稿生成エンジン
- 商品情報、クリック実績、過去投稿データをもとに投稿文を生成する
- AIっぽい定型文を避ける
- 猥談・雑談投稿は自然な口調にする
- 露骨すぎる成人向け表現は禁止
- 未成年を連想させる表現は禁止
- 実在人物・アイドル・芸能人に似せる表現は禁止
- 非同意、盗撮、強制、制服、学生、ロリ、幼い等の危険ワードは禁止
- 投稿文には必ず安全フィルターを通す

3. クリック計測・学習ループ
- Rebrandlyなどのクリック計測データがあれば読み込む
- 投稿ごとのクリック数、投稿時間、カテゴリ、文体、画像有無を記録する
- 成果が良い投稿パターンを重み付けする
- 成果が悪い投稿パターンは重みを下げる
- ただし、自動で極端な方向に寄せない
- 変更幅には上限を設ける

4. 投稿キュー
- いきなり投稿せず、まず投稿候補をqueueに保存する
- AUTO_POST_ENABLED=false の場合は投稿せず、Discordまたはログで確認できるようにする
- AUTO_POST_ENABLED=true の場合のみ投稿する
- DRY_RUN=true の場合は投稿処理を実行せず、投稿予定内容のみ出力する

5. Discord運用コマンド
以下のような運用コマンドを追加または整理する：
- /status 現在の稼働状況を表示
- /pause 自動投稿を停止
- /resume 自動投稿を再開
- /queue 投稿予定一覧を表示
- /approve 投稿候補を承認
- /reject 投稿候補を却下
- /metrics 成果を表示
- /strategy 現在の投稿戦略を表示
- /review 直近の投稿成果から改善案を生成

6. 安全装置
- 1日の最大投稿数を設定できるようにする
- 1時間あたりの最大投稿数を設定できるようにする
- 同じURL、同じ商品、似た文章の連投を防ぐ
- 危険ワード検出時は投稿しない
- 画像生成・画像投稿も安全フィルターを通す
- エラーが連続した場合は自動停止する
- 投稿失敗時はリトライしすぎない
- すべての自動判断をログに残す

7. 設定ファイル
以下のような設定を管理できるようにする：
- AUTO_POST_ENABLED
- DRY_RUN
- MAX_POSTS_PER_DAY
- MAX_POSTS_PER_HOUR
- COOLDOWN_MINUTES
- DEFAULT_PROVIDER
- CATEGORY_WEIGHTS
- SAFETY_STRICTNESS
- DISCORD_NOTIFY_ENABLED
- AI_REVIEW_ENABLED

8. ログ・監視
- 投稿候補生成ログ
- 投稿成功ログ
- 投稿失敗ログ
- 安全フィルターで弾いた理由
- クリック数同期ログ
- AIレビュー結果
- Discord通知ログ
を確認できるようにする。

9. ヘルスチェックAPI
以下のエンドポイントを追加または整理する：
- GET /health
- GET /api/bot/status
- GET /api/bot/queue
- GET /api/bot/metrics
- POST /api/bot/pause
- POST /api/bot/resume
- POST /api/bot/dry-run

10. テスト
最低限、以下のテストを追加する：
- 自動投稿OFFなら投稿されない
- DRY_RUNなら投稿されない
- 危険ワードを含む投稿は弾かれる
- 最大投稿数を超えると投稿されない
- 同一URLの連投が防止される
- キュー生成が正常に動く
- 投稿成功時にログが残る
- 投稿失敗時に安全に停止またはリトライ制限される

実装方針：
- まず既存コードを読み取り、現在の構成を説明する
- 次に実装計画を出す
- その後、必要なファイルを作成・修正する
- 既存機能を壊さない
- TypeScriptの場合は型安全にする
- エラー処理を丁寧に書く
- コメントを適度につける
- 実装後、変更ファイル一覧と使い方を説明する
- 最後に必要な環境変数一覧を出す

重要：
このBotは完全放置のスパムBotではなく、成果データを見ながら安全に改善する半自律運用Botとして設計してください。
初期状態では DRY_RUN=true、AUTO_POST_ENABLED=false にしてください。
本番投稿を開始するには、管理者が明示的に設定を変更する必要があるようにしてください。
